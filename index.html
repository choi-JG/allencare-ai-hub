노하우 팁

AI를 더 잘 활용하기 위한 실용 가이드와 프롬프트 작성법입니다.

"10년 경력의 마케터처럼 답해줘"처럼 역할을 먼저 정의하면 훨씬 전문적이고 상황에 맞는 답변이 나옵니다. AI는 역할에 따라 사용하는 어휘, 깊이, 관점이 달라집니다.

# 예시 프롬프트
당신은 B2B SaaS 마케팅 10년 경력의 전문가입니다.
우리 제품의 LinkedIn 광고 문구 3가지를 작성해주세요.

"3개 항목으로", "표 형식으로", "300자 이내로"처럼 형식을 명시하면 바로 복사해서 쓸 수 있는 결과물이 나옵니다.

# 형식 지정 예시
아래 내용을 불릿 포인트 5개로, 각 항목 50자 이내로 요약해줘.
표 형식으로 정리하되 열은 항목/설명/우선순위로 구성해줘.

"이메일 써줘"보다 "B2B 고객 대상, 계약 갱신 독려 목적, 정중하고 간결하게, 200자 내외"처럼 상황을 설명할수록 정확한 결과가 나옵니다.

# 좋은 프롬프트 = 역할 + 상황 + 과제 + 형식
[역할] 10년 경력 B2B 영업 전문가
[상황] HR SaaS 솔루션 판매, 중소기업 대상
[과제] 신규 리드 첫 콜드 이메일 작성
[형식] 제목 1줄 + 본문 150자 + CTA 1개, 3가지 버전

첫 답변에 만족하지 말고 이어서 수정 요청하세요. "더 간결하게", "공격적인 마케팅 톤으로"처럼 피드백하면 점점 원하는 결과가 됩니다.

# 이어서 수정하는 방법
→ "더 짧게 줄여줘" (절반 길이로)
→ "2번 버전을 더 부드러운 톤으로 바꿔줘"
→ "수치를 더 강조하고 감정적 표현 줄여줘"

AI는 틀린 정보를 자신감 있게 말하기도 합니다. 이를 할루시네이션(Hallucination)이라고 하며, 확인 없이 그대로 공유하거나 보고서에 사용하면 심각한 오류로 이어질 수 있습니다.

⚠ 이렇게 사용하면 위험합니다
•AI는 업무 보조 도구입니다. 최종 검토·승인은 담당자가 직접 수행해야 합니다.
•AI 답변을 그대로 외부에 전달하거나 보고서에 사용하지 마세요.
•중요한 결정에 AI 답변을 단독 근거로 사용하지 마십시오.
•수치·법령·정책 정보는 반드시 원본 자료와 대조 확인하세요.
→AI가 확신 있게 말해도 틀릴 수 있습니다. 최종 판단은 항상 사람이 합니다.

아래 내용을 개인 지침(Custom Instructions)에 등록하거나, 대화 시작 전에 붙여넣고 시작하면 더 신뢰할 수 있는 답변을 받을 수 있습니다.

⚠ 단, 어떤 지침을 사용해도 할루시네이션을 완전히 없애는 건 불가능합니다. 특히 법률·의료·통계 관련 정보는 반드시 원본 출처를 교차 확인하는 습관이 필요합니다.

# 답변 원칙 1. 확실하지 않은 정보는 반드시 "확인된 정보가 없습니다" 또는 "모르겠습니다"라고 말해줘. 추측하거나 그럴듯하게 지어내지 마. 2. 사실 주장을 할 때는 근거나 출처를 함께 제시해줘. 출처가 없거나 불명확하면 추측하지 말고 그렇다고 명시해줘. 3. 답변을 아래 형식으로 구분해줘: - 확인된 사실 - 일반적 해석/추론 - 불확실하거나 추가 검증이 필요한 내용 4. 정확한 근거가 없는 숫자·통계·날짜·링크는 제시하지 마. 만들어낸 URL이나 논문 제목은 절대 쓰지 마. 5. 답변 전에 스스로 검토해서, 근거 없는 단정 표현이나 출처 없는 주장이 있으면 삭제하거나 불확실하다고 표시해줘.

AI 도구 사용 시 유의사항

할루시네이션 주의
AI가 확신 있게 말해도 틀릴 수 있습니다. 그대로 사용하지 말고 최종 판단 및 검토는 반드시 담당자가 직접 합니다.

Projects는 대화 맥락을 저장하는 공간입니다. Custom Instructions에 역할·말투·형식을 한 번만 적어두면 매번 설명 없이도 일관된 답변을 받을 수 있습니다. 업무·학습·개인 등 용도별로 여러 개 만드는 것이 핵심입니다.

# 4단계 세팅법
STEP 01 사이드바 Projects → + 새 프로젝트 클릭
STEP 02 이름·설명 입력 (예: "마케팅팀 업무 / 주간 보고서·카피용")
STEP 03 Custom Instructions — 역할·말투·형식을 한 번만 작성 ← 핵심
STEP 04 테스트 대화로 존댓말·표·상투어 없음 확인
# Projects, 딱 세 줄 요약
01 — Projects는 대화 맥락을 저장하는 공간이다.
02 — Custom Instructions + 참조 파일 = 나만의 AI 비서.
03 — 업무·학습·개인, 용도별로 여러 개 만들어라.

역할 1줄로 시작하고, 답변 규칙 3~5개, 금지어, 자주 하는 작업 순서로 작성하세요. 길게 쓸수록 오히려 성능이 떨어집니다. 20줄 이내를 권장합니다.
